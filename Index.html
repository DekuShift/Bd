Fundamentos

01 Conceitos básicos antes de começar

O que é PHP?

PHP é uma linguagem que roda no servidor (no computador que hospeda o site). O usuário nunca vê o código PHP — ele só vê o resultado em HTML. Todo código PHP fica entre as tags <?php e ?>.

O que é MySQL?

MySQL é um banco de dados — pense nele como uma planilha do Excel, mas muito mais poderosa. Os dados ficam em tabelas com linhas (registros) e colunas (campos).

O que é XAMPP?

XAMPP é um programa que instala o Apache (servidor web) e o MySQL no seu computador, permitindo rodar PHP localmente, sem precisar de hospedagem.

Variáveis em PHP

Toda variável começa com $. É diferente de JavaScript ou Python.

exemplo.phpPHP
$nome = "Maria";          // string (texto)
$idade = 20;             // número inteiro
echo $nome;              // mostra "Maria" na tela

banco.sql

02 banco.sql — Criando o banco de dados

Esse arquivo contém os comandos SQL que criam o banco e a tabela. Você importa ele uma única vez no phpMyAdmin.

banco.sqlSQL
CREATE DATABASE crud_estudo;   -- Cria um banco chamado "crud_estudo"

USE crud_estudo;               -- Diz "use esse banco daqui pra frente"

CREATE TABLE usuarios(
    id    INT AUTO_INCREMENT PRIMARY KEY,  -- número único automático para cada linha
    campo1 VARCHAR(100),                   -- texto de até 100 caracteres
    campo2 VARCHAR(100),
    campo3 VARCHAR(100)
);
📌

AUTO_INCREMENT significa que o MySQL cria automaticamente um número único para cada registro (1, 2, 3...). Você nunca precisa informar o ID manualmente. PRIMARY KEY significa que esse campo identifica cada linha de forma única — dois registros jamais terão o mesmo ID.

conexao.php

03 conexao.php — A ponte com o banco

Esse arquivo é o mais importante. Ele cria a conexão com o MySQL e é incluído em todos os outros arquivos PHP.

conexao.phpPHP
<?php
// ↑ tag que abre o bloco PHP

$conexao = new mysqli(
    "localhost",   // endereço do servidor MySQL (seu próprio computador)
    "root",        // usuário padrão do XAMPP
    "",            // senha (vazia por padrão no XAMPP)
    "crud_estudo"  // nome do banco que criamos no banco.sql
);

// Verifica se deu erro na conexão:
if($conexao->connect_error){
    die("Erro na conexão");  // die() para tudo e mostra a mensagem
}
?>

Por que incluir em todos os arquivos?

Usando include "conexao.php" no início de cada arquivo PHP, você não precisa repetir esse código. Se um dia precisar mudar a senha ou o nome do banco, altera só nesse arquivo.

⚡

$conexao->algo — o símbolo -> em PHP é como o ponto em outras linguagens. Significa "acesse a propriedade ou método do objeto". Aqui, $conexao é um objeto da classe mysqli.

O atributo name do input é fundamental. É ele que o PHP usa para recuperar o valor: $_POST['campo1']. Se o name for diferente, o PHP não acha o dado.

05 cadastrar.php — Salvando dados (CREATE)

Recebe os dados do formulário via $_POST e os insere no banco com INSERT INTO.

cadastrar.phpPHP
<?php

include "conexao.php";  // importa a conexão com o banco

// $_POST['campo1'] = o que o usuário digitou no input name="campo1"
$campo1 = $_POST['campo1'];
$campo2 = $_POST['campo2'];
$campo3 = $_POST['campo3'];

// Monta o comando SQL para inserir um novo registro na tabela
$sql = "INSERT INTO usuarios(campo1, campo2, campo3)
         VALUES('$campo1', '$campo2', '$campo3')";

// Envia o comando para o MySQL e verifica se funcionou
if($conexao->query($sql)){
    echo "Cadastro realizado!";
}else{
    echo "Erro ao cadastrar";
}

?>

listar.php

06 listar.php — Exibindo dados (READ)

Busca todos os registros do banco e os exibe na tela, um por um, com links para editar e excluir.

listar.phpPHP
<?php

include "conexao.php";

// SELECT * = "pegue TUDO" FROM usuarios = "da tabela usuarios"
$sql       = "SELECT * FROM usuarios";
$resultado = $conexao->query($sql);  // executa e guarda o resultado

// fetch_assoc() pega uma linha do resultado como um array associativo
// O while continua enquanto houver linhas para ler
while($dados = $resultado->fetch_assoc()){

    echo "ID: "      . $dados['id']     . "<br>";
    echo "Campo 1: " . $dados['campo1'] . "<br>";
    echo "Campo 2: " . $dados['campo2'] . "<br>";
    echo "Campo 3: " . $dados['campo3'] . "<br>";

    // Link para excluir: passa o ID do registro na URL (?id=X)
    echo "<a href='excluir.php?id=" . $dados['id'] . "'>Excluir</a>";

    // Link para editar: também passa o ID na URL
    echo "<a href='editar.php?id="   . $dados['id'] . "'>Editar</a><br>";

    echo "<hr>";  // linha separadora entre registros
}

?>
💡

O ponto . em PHP concatena (junta) strings. "Olá " . "mundo" resulta em "Olá mundo". Isso é diferente de JavaScript, onde se usa +, ou Python, onde também se usa +.

07 editar.php — Formulário de edição

Recebe o id pela URL, busca os dados atuais no banco e preenche um formulário para o usuário editar.

editar.phpPHP + HTML
<?php

include "conexao.php";

// $_GET['id'] pega o valor "id" que veio na URL: editar.php?id=5
$id        = $_GET['id'];

$sql       = "SELECT * FROM usuarios WHERE id = $id"; // busca só esse registro
$resultado = $conexao->query($sql);
$dados     = $resultado->fetch_assoc();  // pega a linha (só 1 aqui)

?>

<!-- O PHP fechou, agora vem HTML normal + PHP embutido nos values -->
<form action="atualizar.php" method="POST">

    <!-- Campo oculto: carrega o ID sem aparecer na tela -->
    <input type="hidden" name="id"
           value="<?php echo $dados['id']; ?>">

    Campo 1:
    <input type="text" name="campo1"
           value="<?php echo $dados['campo1']; ?>">

    Campo 2:
    <input type="text" name="campo2"
           value="<?php echo $dados['campo2']; ?>">

    Campo 3:
    <input type="text" name="campo3"
           value="<?php echo $dados['campo3']; ?>">

    <button type="submit">Salvar</button>
</form>
💡

O type="hidden" guarda o ID do registro no formulário sem exibir ao usuário. É necessário para que o atualizar.php saiba qual linha do banco deve ser alterada.

atualizar.php

08 atualizar.php — Salvando a edição (UPDATE)

Recebe os dados editados pelo formulário do editar.php e atualiza o registro no banco.

atualizar.phpPHP
<?php

include "conexao.php";

$id     = $_POST['id'];      // pega o ID que veio do campo hidden
$campo1 = $_POST['campo1'];
$campo2 = $_POST['campo2'];
$campo3 = $_POST['campo3'];

// UPDATE: atualiza registros existentes
// SET: define quais colunas mudam e para qual valor
// WHERE: limita a atualização só ao registro com esse ID
$sql = "UPDATE usuarios
         SET campo1 = '$campo1',
             campo2 = '$campo2',
             campo3 = '$campo3'
         WHERE id = $id";

if($conexao->query($sql)){
    echo "Registro atualizado!";
}else{
    echo "Erro ao atualizar.";
}

?>
⚠️

O WHERE id = $id é crítico. Sem ele, o UPDATE atualizaria todos os registros da tabela de uma vez. Sempre use WHERE ao atualizar ou excluir.

excluir.php

09 excluir.php — Apagando dados (DELETE)

Recebe o ID pela URL e remove o registro correspondente do banco.

excluir.phpPHP
<?php

include "conexao.php";

$id  = $_GET['id'];   // pega o ID da URL: excluir.php?id=3

// DELETE FROM: exclui da tabela o registro com esse ID específico
$sql = "DELETE FROM usuarios WHERE id = $id";

if($conexao->query($sql)){
    echo "Registro excluído!";
}else{
    echo "Erro ao excluir";
}

?>
Segurança

10 Alertas de segurança

Este projeto é ótimo para aprendizado, mas tem vulnerabilidades sérias para uso em produção (internet real). São importantes de conhecer:

🚨

SQL Injection: O código insere variáveis diretamente no SQL ("WHERE id = $id"). Um atacante pode digitar código SQL no lugar de um valor normal e destruir ou vazar o banco. A solução é usar Prepared Statements (próximo passo nos seus estudos).

🚨

XSS (Cross-Site Scripting): Dados do usuário são exibidos sem filtragem. Alguém pode salvar <script>alert('hackeado')</script> como valor. Use htmlspecialchars($variavel) ao exibir dados do banco na tela.

⚡

Para estudos locais com XAMPP, essas vulnerabilidades não são problema. Elas só importam quando o sistema for para a internet. O objetivo agora é entender a lógica do CRUD!
